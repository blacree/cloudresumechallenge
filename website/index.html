Enviroment Configuration for EC2 Autoscaling on AWS


This project which uses Terraform consists of two parts. The first part involves configuring an autoscaling group that deploys multiple EC2 instances of an AMI to serve a web page. The second part involves the use of Terraform workspaces to dynamically control the type of infrastrcture environment that is deployed. The environments configured include default, staging and production. This helped to control the properties of the infrastrcture that is deployed on each separate environment. some of these properties include; the AMI Id, EC2 instance type, VPC Id, the Subnets selected for the instances, and the desired, minimum and maximum no of Ec2 instances to be spawned in the auto scaling group.
